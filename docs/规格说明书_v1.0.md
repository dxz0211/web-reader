## 1. 业务目标

### 1.1 一句话定位

一个基于浏览器的沉浸式阅读创作平台。创作者为文本段落配背景图与音乐、编排时间轴，生成 `.ims` 文件；读者加载该文件后获得自动播放、音画同步的沉浸式阅读体验。

### 1.2 核心用户故事

| 角色 | 故事 |
|------|------|
| 创作者 | 打开编辑器 → 输入文本段落 → 为每段选背景图/音乐 → 设置时间 → 保存 `.ims` |
| 读者 | 打开播放器 → 选择 `.ims` → 点击播放 → 自动显示文字/切换图片/播放音乐 |

### 1.3 答辩演示场景（验收终极标准）

> 编辑器从零创作一个 3 段作品 → 保存为 `demo.ims` → 播放器加载 → 自动播放，文字/图片/音乐按时间轴正确切换。

## 3. 数据格式规范

### 3.1 `.ims` 文件 JSON Schema

```json
{
  "version": "1.0",
  "metadata": {
    "title":        {"type": "string", "required": true,  "desc": "作品标题"},
    "author":       {"type": "string", "required": true,  "desc": "原文作者"},
    "creator":      {"type": "string", "required": true,  "desc": "编辑器操作者"},
    "description":  {"type": "string", "required": false, "desc": "作品简介"}
  },
  "assets": {
    "images": {
      "<alias>": "<relative_path>"
    },
    "audios": {
      "<alias>": "<relative_path>"
    }
  },
  "timeline": [
    {
      "segment_id":        {"type": "int",   "required": true, "desc": "段落序号，从1开始"},
      "text":              {"type": "string","required": true, "desc": "该段显示的文字"},
      "start_time":        {"type": "float", "required": true, "desc": "开始时间(秒)"},
      "duration":          {"type": "float", "required": true, "desc": "持续时长(秒)"},
      "background":        {"type": "string","required": true, "desc": "对应 assets.images 的 key"},
      "background_music":  {"type": "string","required": false,"desc": "对应 assets.audios 的 key"},
      "transition_effect": {"type": "string","required": false,"desc": "fade_in | cross_fade | none"}
    }
  ]
}
```

### 3.2 强制规则

1. `assets.images` 的 key 与 `timeline[].background` 的值必须一一对应
2. `assets.audios` 的 key 与 `timeline[].background_music` 的值必须一一对应
3. `timeline[]` 按 `start_time` 升序排列
4. 相邻段落无时间空隙：下一段 `start_time` = 上一段 `start_time + duration`
5. 所有文件路径均为相对于项目根目录的相对路径

### 3.3 存档文件格式（`savedata/save_{slot}.json`）

```json
{
  "slot": 0,
  "filename": "sample.ims",
  "segment_index": 5,
  "timestamp": "2025-07-14 15:30:00"
}
```

## 4. 技术约束

| 约束项 | 规定 |
|--------|------|
| 后端框架 | Python Flask（≤ 100 行） |
| 前端技术 | 纯 HTML + CSS + Vanilla JavaScript，**不使用任何前端框架** |
| 数据存储 | 文件系统（JSON），**不使用数据库** |
| 浏览器 | Chrome 90+ / Edge 90+ |
| Python | 3.8+ |
| 依赖 | 仅 Flask（`requirements.txt` 一行） |

**不用前端框架的原因**：
- 本项目 JS 逻辑约 300 行，框架学习成本远超实现成本
- 原生 `<audio>`、`fetch()`、CSS `transition` 已完全覆盖需求
- 零构建工具，无需 npm/webpack，保存即生效
- 团队成员可在 30 分钟内理解全部前端代码

```
│ 描述     │ │ 起始 [__] 持续  │ │             │

│ [打开]   │ │ ...             │ │ [+添加图片] │
│          │ └─────────────────┘ │ [+添加音频] │

└──────────┴─────────────────────┴─────────────┘
```

**交互规则**：
- 点击「+ 添加段落」：末尾追加空白段落卡片（编号自动递增）
- 点击「✕ 删除」：删除该卡片（至少保留 1 段）
- 点击「💾 保存」：收集所有表单数据 → 构造 JSON → `POST /api/save`
- 点击「📂 打开」：`GET /api/files` → 弹窗选文件 → `GET /api/load` → 填充表单
- 每段默认 `transition_effect` 为 `"cross_fade"`

### 6.1.1 段落属性与播放效果的映射关系

编辑器中的每个设置项，直接影响播放器中的呈现效果，映射如下：

| 编辑器设置项 | .ims 字段 | 播放器呈现效果 |
|-------------|-----------|---------------|
| 段落文字输入框 | `timeline[N].text` | 播放时在屏幕中央显示该文字 |
| 起始时间输入框 | `timeline[N].start_time` | 播放器在此时刻开始显示该段落 |
| 持续时间输入框 | `timeline[N].duration` | 该段文字和背景图持续显示这么久 |
| 背景图下拉选择 | `timeline[N].background` | 播放时背景切换为该图片 |
| 背景音乐下拉选择 | `timeline[N].background_music` | 播放时循环播放该音频 |
| 切换效果下拉选择 | `timeline[N].transition_effect` | 图片切换时的动画效果 |

**切换效果枚举**：
- `none`：图片直接替换，无动画
- `fade_in`：新图片从透明渐变到完全显示
- `cross_fade`：旧图片淡出的同时新图片淡入

**播放器实现要点**：
- 图片切换使用 CSS `transition: opacity 0.8s ease-in-out`
- 音乐切换使用 HTML5 `<audio>` 元素的 `src` 属性替换
- 时间轴检查频率为每 200ms 一次，确保切换及时
- 每段的 duration 结束后，自动进入下一段的 start_time

### 6.2 播放器页面（player.html）

**布局**：

```
┌────────────────────────────────────────┐
│  [标题]              第 N 段 / 共 M 段  │
│  [作者]                                │
│                                        │
│                                        │
│         ┌─────────────────────┐        │
│         │ 撑着油纸伞，独自      │        │
│         │ 彷徨在悠长、悠长     │        │
│         │ 又寂寥的雨巷         │        │
│         └─────────────────────┘        │
│                                        │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ ▶/⏸ │ 下一段 │ 存档 │ 读档 │ 回想│  │
│  │        音量: [═══──]             │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

**状态机**：

```
[待加载] ──选文件加载──→ [已就绪] ──▶──→ [播放中]
                            ↑              │  │
                            │         暂停 ╱   ╲ 段落到末尾
                            │         ↓          ↓
                            │    [已暂停]    [已结束]
                            │         │          │
                            │      ▶ 继续    重新播放
                            │         │          │
                            └─────────┴──────────┘
```

**核心算法**（时间轴判断，每 200ms 执行一次）：

```
currentTime = 从播放开始到现在经过的秒数
currentSegment = timeline.find(seg =>
    seg.start_time <= currentTime &&
    currentTime < seg.start_time + seg.duration
)
if currentSegment.segment_id != displayedSegmentId:
    更新文字 → 切换背景图(CSS淡入淡出) → 切换音乐 → 更新进度
```

### 6.3 存档/读档交互（播放器内）

- **存档弹窗**：10 个槽位按钮，空槽位显示「[空]」，已有存档显示时间戳 + 文件名 + 所在段落
- **读档弹窗**：同存档弹窗，空槽位不可点击，点击已有存档 → 加载 → 从保存的 `segment_index` 重新播放

## 7. API 接口规范

### 7.1 页面路由

| 方法 | 路径 | 返回 |
|------|------|------|
| GET | `/` | 重定向至 `/player` |
| GET | `/editor` | `editor.html` |
| GET | `/player` | `player.html`（可带 `?file=xxx.ims` 自动加载） |
| GET | `/help` | `help.html` |

### 7.2 数据 API

| 方法 | 路径 | 请求参数 | 返回 | 错误 |
|------|------|----------|------|------|
| GET | `/api/files` | 无 | `["sample.ims", ...]` | — |
| GET | `/api/load` | `?file=xxx.ims` | `.ims` 完整 JSON | `{"error":"file not found"}`, 404 |
| POST | `/api/save` | Body: .ims JSON | `{"status":"ok","filename":"xxx.ims"}` | `{"error":"invalid json"}`, 400 |
| GET | `/api/assets/images` | 无 | `["lantern.jpg", ...]` | — |
| GET | `/api/assets/audios` | 无 | `["xxx.mp3", ...]` | — |

### 7.3 存档 API

| 方法 | 路径 | 请求参数 | 返回 |
|------|------|----------|------|
| POST | `/api/save_progress` | Body: `{"slot":0, "filename":"...", "segment_index":5}` | `{"status":"ok"}` |
| GET | `/api/load_progress` | `?slot=0` | `{"filename":"...", "segment_index":5, "timestamp":"..."}` |
| GET | `/api/list_progress` | 无 | `[{"slot":0, "filename":"...", "timestamp":"..."}, ...]` |
